## 1. 任务组成

从 [config.py:1826-1836](src/openpi/training/config.py#L1826-L1836) 可以看到 9 个任务：

| 序号 | 任务名称 | 类型 | 说明 |
|------|----------|------|------|
| 1 | Pouring workpieces (倾倒工件) | 单臂 | 仅使用右手倾倒工件 |
| 2 | Opening a door (开门) | 单臂 | 仅使用右手开门 |
| 3 | Scooping popcorn (挖爆米花) | 单臂 | 仅使用右手挖爆米花 |
| 4 | Carrying a pot (端锅) | 双臂 | 双手同时使用 |
| 5 | Grabbing toys (抓玩具) | 双臂 | 根据指令使用左手或右手 |
| 6 | Supermarket item retrieval (超市取物) | 双臂 | 根据指令使用左手或右手 |
| 7 | Supermarket restocking (超市补货) | 双臂 | 根据指令使用左手或右手 |
| 8 | Packages sorting (包裹分拣) | 单臂+腰部 | 需要腰部转动 |
| 9 | Arranging the table (整理桌面) | 双臂 | 双手同时使用 |

### 任务特点分析

| 任务类型 | 数量 | 特点 |
|----------|------|------|
| 单臂任务 | 4 个 | 相对简单，主要关注精细操作 |
| 双臂任务 | 4 个 | 需要双手协调，难度中等 |
| 腰部任务 | 1 个 | 需要腰部转动，动作维度更多 |

## 2. 评分方式

基于典型的机器人学习比赛模式，推测评分方式如下：

### 主要评分公式

```
最终得分 = 任务成功率 × 效率系数
```

**其中：**
- **任务成功率**：所有任务的平均成功完成率
- **效率系数**：考虑推理时间、动作步数等因素的调整系数

### 具体评分指标

#### 2.1 二进制成功判定
```
- 0 = 失败（任务未完成）
- 1 = 成功（任务完全完成）
```

#### 2.2 部分成功度评分（0-1 连续值）
```
基于目标达成的百分比：
- 工件倒入了 50% → 0.5 分
- 门开了一半 → 0.5 分
- 完全完成 → 1.0 分
```

#### 2.3 效率奖励/惩罚

| 因素 | 奖励/惩罚 | 说明 |
|------|-----------|------|
| 更短的执行时间 | 奖励 | 快速完成任务 |
| 更少的动作步数 | 奖励 | 更高效的策略 |
| 超时 | 惩罚 | 超过时间限制 |
| 动作平滑度 | 可能奖励 | 更自然的动作 |

### 各任务权重（推测）

所有任务可能权重相等，即：
```
总成功率 = (任务1成功率 + 任务2成功率 + ... + 任务9成功率) / 9
```

```bash

```

### 3.3 WebSocket 服务协议

服务器使用 [websocket_policy_server.py](src/openpi/serving/websocket_policy_server.py) 提供服务。

#### 通信流程

```
1. 连接建立 → 服务器发送元数据
2. 客户端发送观测数据 (obs)
3. 服务器执行推理 → 返回动作 (actions)
4. 重复步骤 2-3 直到任务结束
```

#### 观测数据格式

```python
obs = {
    "observation/images/top_head": np.ndarray,      # 顶部视角图像 (uint8, HWC)
    "observation/images/hand_left": np.ndarray,     # 左手视角图像 (uint8, HWC)
    "observation/images/hand_right": np.ndarray,    # 右手视角图像 (uint8, HWC)
    "observation/state": np.ndarray,                 # 机器人状态 (32维, float32)
    "prompt": "任务指令...",                          # 自然语言指令 (string)
}
```

#### 动作输出格式

```python
action = {
    "actions": np.ndarray,  # (action_horizon, action_dim) - 预测的动作序列
    "state": np.ndarray,    # 当前状态
    "policy_timing": {
        "infer_ms": float   # 推理耗时 (毫秒)
    }
}
```

## 5. 针对挑战赛的优化建议

### 5.1 任务特定优化

#### 任务 8（Packages sorting）- 需要腰部动作

在 [policy.py:73-89](src/openpi/policies/policy.py#L73-L89) 中已有特殊处理：

```python
def post_process(self, obs: dict, outputs: dict) -> dict:
    task_name_requiring_waist = ["sorting_packages", "sorting_packages_continuous"]
    task_name = obs.get("task_name", None)

    if task_name is None:
        return outputs

    if task_name not in task_name_requiring_waist:
        # 截断腰部动作（不需要的任务）
        outputs["actions"] = outputs["actions"][:, :16]
    else:
        # 保留腰部动作，但可能优化
        raw_state = obs.get("state", None)
        if raw_state is not None:
            # 冻结前几个腰部维度，只使用最后一个
            outputs["actions"][:, 16:20] = raw_state[16:20]

    return outputs
```

**进一步优化建议：**
```python
def _optimize_waist_actions(self, actions):
    """针对分拣任务优化腰部动作"""
    # 腰部动作维度：16-19
    waist_actions = actions[:, 16:20]

    # 应用平滑
    for dim in range(4):
        waist_actions[:, dim] = self._exponential_smoothing(waist_actions[:, dim], alpha=0.9)

    # 限制动作幅度
    waist_actions = np.clip(waist_actions, -0.8, 0.8)

    actions[:, 16:20] = waist_actions
    return actions

def _exponential_smoothing(self, sequence, alpha=0.8):
    """指数平滑"""
    smoothed = np.zeros_like(sequence)
    smoothed[0] = sequence[0]
    for i in range(1, len(sequence)):
        smoothed[i] = alpha * sequence[i] + (1 - alpha) * smoothed[i-1]
    return smoothed
```

**特点：**
- 需要双手协调

2. 考虑增加双臂任务的数据权重

挑战赛配置中使用了丰富的 prompt 注入：

```python
    ("简洁描述", 0.3),
    ("分步描述", 0.2),
```

2. **包含更多细节信息**
```python

```

3. **描述关键子步骤**

4. **使用模板化 prompt**
```python

```

### 5.4 集成学习策略

```python
# 训练多个不同配置的模型
configs = [
    (30, 30, "downsample"),
    (40, 20, "query"),
    (25, 25, "attention"),
]

# 推理时集成
def ensemble_infer(obs, models):
    predictions = []
    for model in models:
        pred = model.infer(obs)
        predictions.append(pred["actions"])

    # 加权平均
    final_actions = np.mean(predictions, axis=0)
    return {"actions": final_actions}
```
